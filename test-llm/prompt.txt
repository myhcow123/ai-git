## AI-Git Usage Guide

AI-Git is a code management system designed for AI models. Here are the steps to use it:

### Important Notes

**AI-Git executable location**: `/home/mychow/Documents/cs/ai-git/ai-git`

**All ai-git commands need to use the full path**, for example:
```bash
/home/mychow/Documents/cs/ai-git/ai-git daemon start
/home/mychow/Documents/cs/ai-git/ai-git search "function_name"
```

### 1. Background Service Management (Required)

**Important: Ensure the background service is running before all operations!**

```bash
# Check service status (recommended to execute first)
/home/mychow/Documents/cs/ai-git/ai-git daemon status

# If service is not running, start the service
/home/mychow/Documents/cs/ai-git/ai-git daemon start

# Stop service
/home/mychow/Documents/cs/ai-git/ai-git daemon stop
```

### 2. Initialize Project
```bash
# Initialize in project directory
cd /path/to/project
/home/mychow/Documents/cs/ai-git/ai-git init
```

### 3. Query Code

**Prefer using AI-Git features over cat command!**

```bash
# Search symbols
/home/mychow/Documents/cs/ai-git/ai-git search "function_name"

# Query symbol details
/home/mychow/Documents/cs/ai-git/ai-git symbol "symbol_name"

# Read symbol code (Note: read is for symbols, not file paths)
/home/mychow/Documents/cs/ai-git/ai-git read "symbol_name"

# Read specific lines of a file (format: file:line-start-line-end)
/home/mychow/Documents/cs/ai-git/ai-git read "main.go:10-20"

# View project overview
/home/mychow/Documents/cs/ai-git/ai-git overview

# View project status
/home/mychow/Documents/cs/ai-git/ai-git status
```

### 4. Modify Code
```bash

/home/mychow/Documents/cs/ai-git/ai-git insert "symbol_name" --code "// comment"

# Replace code
/home/mychow/Documents/cs/ai-git/ai-git replace "symbol_name" --with "new code"

# Delete code
/home/mychow/Documents/cs/ai-git/ai-git delete "symbol_name"
```

### 5. Other Features

AI-Git provides task management functionality to help you track and manage task progress.


### 2. View Tasks
```bash

# View specific task details
```

### 3. Update Task Progress
```bash

# Update task status
```

### 4. Task Workflow

**Recommended task management process:**

1. **Analyze task** - Break down complex tasks into multiple subtasks
2. **Create tasks** - Create independent task records for each subtask
3. **Execute tasks** - Execute each task step by step according to priority
4. **Update progress** - Regularly update task progress
5. **Complete tasks** - Mark tasks as completed when finished

## Error Handling Best Practices

### 1. Command Execution Failure
- Check if command path is correct
- Check if parameter format is correct
- Read error message to understand failure reason
- Try different approaches

### 2. Compilation Error
- Carefully read error message
- Check if imports are used
- Check if syntax is correct
- Recompile after fixing

### 3. AI-Git Functionality Unavailable
- Check if background service is running
- Check if project is initialized
- Use full path to execute commands
